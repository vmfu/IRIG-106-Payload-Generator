What is IRIG 106?

IRIG 106 (Inter-Range Instrumentation Group Standard 106) is an international standard for recording and transmitting telemetry data in aerospace, defense, and scientific applications. Chapter 4 defines the PCM (Pulse Code Modulation) format ‚Äî a digital representation of analog and discrete signals.

The generator allows you to create binary data packets (.bin), TMATS metadata (.tmats), and reports for testing telemetry processing systems, such as:

  • Ground-based data reception stations
  • Flight Data Recorders (FDR)
  • Launch Vehicle Telemetry Processing Systems
  • Spacecraft and drones

üîÑ Subcommutation (Subcommutation)

Why is Subcommutation Needed?

Real rockets have hundreds of parameters, but not all need transmitting every frame. For example:

  • Fast data (100 Hz): Accelerations, chamber pressure, turbopump RPM
  • Slow data (10 Hz): Temperatures, battery voltages, valve positions

Subcommutation allows packing slow data into one slot, transmitting them in rotation. This saves frame width by up to 10x!

Example: Two-Level Subcommutation

The demo configuration uses nested structure:

		SUBCOMCONTAINER (Level 1: 9 slots, update frequency = 100 Hz / 9 ‚âà 11 Hz):
		  ‚îú‚îÄ Slot 0: BATTERYVOLTAGEA       ‚Üê Battery A voltage (updates every 9 frames)
		  ‚îú‚îÄ Slot 1: BATTERYVOLTAGEB       ‚Üê Battery B voltage
		  ‚îú‚îÄ Slot 2: TANKFUELTEMP          ‚Üê Fuel tank temperature
		  ‚îú‚îÄ Slot 3: TANKOXTEMP            ‚Üê Oxidizer temperature
		  ‚îú‚îÄ Slot 4: IMUTEMPX              ‚Üê IMU (inertial unit) temperature
		  ‚îú‚îÄ Slot 5: DEEPSUBCOMIDCNT       ‚Üê Level 2 subcommutation ID
		  ‚îú‚îÄ Slot 6: DEEPSUBCOMCONTAINER   ‚Üê Nested subcommutation!
		  ‚îÇ   ‚îî‚îÄ‚îÄ (Level 2: 4 slots, frequency = 100 Hz / (9√ó4) ‚âà 2.8 Hz):
		  ‚îÇ       ‚îú‚îÄ VALVE1POS             ‚Üê Valve 1 position
		  ‚îÇ       ‚îú‚îÄ VALVE2POS             ‚Üê Valve 2 position
		  ‚îÇ       ‚îú‚îÄ VALVE3POS             ‚Üê Valve 3 position
		  ‚îÇ       ‚îî‚îÄ VALVE4POS             ‚Üê Valve 4 position
		  ‚îú‚îÄ Slot 7: RESERVEDSLOT6         ‚Üê Reserved slot (for future expansion)
		  ‚îî‚îÄ Slot 8: CHECKSUMSUB           ‚Üê Subcommutation checksum

‚úÖ Result: Instead of 13 words, only 1 word + ID-counter is used! Savings: 92% bandwidth for slow parameters.

How to Decode Subcommutation?

Receiver-side algorithm:

		1. Read SUBCOMIDCNT (slot counter 0-8)
		2. Read value from SUBCOMCONTAINER
		3. Write value to corresponding variable:
		   if (SUBCOMIDCNT == 0) ‚Üí BATTERYVOLTAGEA = decoded_value;
		   if (SUBCOMIDCNT == 1) ‚Üí BATTERYVOLTAGEB = decoded_value;
		   ...
		   if (SUBCOMIDCNT == 6) ‚Üí {
			 read DEEPSUBCOMIDCNT;
			 decode nested subcommutation;
		   }
		4. Next frame: SUBCOMIDCNT++ (mod 9)

üìÑ TMATS ‚Äî Telemetry Attributes Transfer Standard

What is TMATS?

TMATS is a text file containing telemetry metadata describing the data structure. It allows the receiver to automatically decode binary data without manual configuration.

Example TMATS entry:

		G\DSI-1:IRIG 106 Chapter 4 PCM Demo;
		P-1\DLN:ROCKETALTITUDE;
		P-1\TF:IEEE754;
		P-1\DPL:64;
		P-1\MFMT-1-1:1;
		P-1\TCLK:100;
		P-1\MF\N:47;

Explanation:

  • G\DSI-1 ‚Äî System description
  • P-1\DLN ‚Äî Parameter name (Data Link Name)
  • P-1\TF ‚Äî Data format (Transfer Format)
  • P-1\DPL ‚Äî Length in bits (Data Path Length)
  • P-1\MFMT-1-1 ‚Äî Position in Major Frame
  • P-1\TCLK ‚Äî Sampling frequency (Hz)
  • P-1\MF\N ‚Äî Number of words in frame

How to Use TMATS?

After generating .bin and .tmats files:

  1. Load .tmats into processing software (e.g., IRIG 106 DLL, FlightGear, ParaView)
  2. The software automatically reads the data structure
  3. Open .bin ‚Äî parameters will be decoded automatically!

‚úÖ Compatibility: The generator creates TMATS according to IRIG 106-19 (2019 edition), compatible with IRIG 106-11, 106-15.

üîÅ Loopback Testing ‚Äî Data Integrity Verification

What Does Loopback Check?

After generating binary data, the generator automatically decodes it back and compares with original values. This guarantees correct encoding/decoding.

Checked aspects:

  • Floating-point number encoding accuracy (IEEE754, DOUBLE, MIL1750A)
  • Byte order correctness (Big-Endian / Little-Endian)
  • Bit order correctness (MSB-first / LSB-first)
  • Subcommutation decoding correctness
  • Bit error detection (if error injection enabled)

Interpreting Loopback Results

In the generation console you will see:

		‚úÖ Loopback Test: SYNCPATTERN1 ‚Üí Original: 60304, Decoded: 60304, Error: 0.000%
		‚úÖ Loopback Test: ROCKETALTITUDE ‚Üí Original: 45678.123, Decoded: 45678.123, Error: 0.000%
		‚ö†Ô∏è Loopback Test: CHAMBERPRESSURE1 ‚Üí Original: 8.5, Decoded: 8.500001, Error: 0.0001% (IEEE754 rounding)

‚ùå Common Errors and How to Fix Them

Error: "Circular dependency detected"

Cause: Parameter A depends on B, and B depends on A.

Solution:

		Incorrect:
		  THRUST: formula ‚Üí depends on ACCELERATION
		  ACCELERATION: formula ‚Üí depends on THRUST  ‚Üê CIRCULAR DEPENDENCY!

		Correct:
		  CHAMBERPRESSURE: sine (independent)
		  THRUST: formula ‚Üí depends on CHAMBERPRESSURE
		  ACCELERATION: formula ‚Üí depends on THRUST

Error: "Parameter ID not found in formula"

Cause: Formula references a non-existent parameter.

Solution: Verify that the ID in DependsOn exactly matches the parameter ID (case-sensitive!).

		Incorrect:
		  Formula: p.RocketVelocity * 2   ‚Üê "RocketVelocity" not found!
		  DependsOn: ROCKETVELOCITY

		Correct:
		  Formula: p.ROCKETVELOCITY * 2   ‚Üê Exact match
		  DependsOn: ROCKETVELOCITY

Error: "NaN detected in IEEE754 encoding"

Cause: Division by zero or Math.sqrt(-1) in formula.

Solution: Add checks to formula:

		Incorrect:
		  Formula: p.THRUST / p.MASS      ‚Üê If MASS=0, results in NaN!

		Correct:
		  Formula: p.MASS > 0 ? p.THRUST / p.MASS : 0

Problem: Loopback Shows Error > 1%

Cause: Incorrect data type or bit count selection.

Solution:

  • For GPS coordinates use DOUBLE (64 bits), not IEEE754 (32 bits)
  • For temperatures ¬±200¬∞C: SB 16 bits is sufficient
  • For counters: use UB, not IEEE754

Problem: Subcommutation Decoded Incorrectly

Cause: ID-counter (SUBCOMIDCNT) not synchronized with container.

Solution: Ensure:

  1. ID-counter has behavior: counter
  2. min=0, max=(number of slots - 1)
  3. ID-counter positioned before container in Major Frame
		Correct order:
		  Word 25: SUBCOMIDCNT (counter, 0-8)
		  Word 26: SUBCOMCONTAINER (9 slots)
